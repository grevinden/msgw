# 🎉 MessageCenter !

Твой личный **шлюз сообщений** с суперсилой: сообщения не теряются, даже если ты отключился! 🔥

## 💡 Что тут вообще происходит?

- Клиенты стучатся через **WebSocket** (живой канал туда‑сюда) или кидают сообщения через **HTTP POST** (для серверов).
- Сервер **запоминает** все важные сообщения (`notify`) на диске – перезагрузился? Не беда, данные останутся! 💾
- Когда приходит новичок, ему **сразу отправляют всю историю** неподтверждённых сообщений.
- А когда ты говоришь **«всё, получил»** (кидаешь `receipt`), сообщение удаляется из хранилища и больше не надоедает новым клиентам.
- Каждое сообщение **улетает всем**, кто сейчас в чате (broadcast). 🚀

# 📦 Типы пакетов (они же – сообщения)

Все пакеты – это JSON. У каждого есть `uuid` (уникальный номер) и `payload` – содержимое.

## 1️⃣ send – «Эй, всем привет!» 👋

```json
{
  "uuid": "123e4567-e89b-12d3-a456-426614174000",
  "payload": {
    "typ": "send",
    "top": "новости",
    "mes": "Сервер перезапущен!"
  }
}
```

- **Что значит**: обычное сообщение, которое нужно доставить всем и получить подтверждение.
- **Что делает сервер**: сохраняет его в надёжное место 📦, а затем рассылает всем подключённым.
- **Что надо сделать тебе**: когда получишь такое – обработай и пришли ответ `done` с тем же `uuid`.

## 2️⃣ done – «Всё, принял, спасибо!» ✅

```json
{
  "uuid": "123e4567-e89b-12d3-a456-426614174000",
  "payload": {
    "typ": "done"
  }
}
```

- **Что значит**: квитанция об успешной обработке.
- **Что делает сервер**: удаляет сообщение из базы (больше никому не отправит) и рассылает `done` всем (чтобы все знали, что задача решена).

## 3️⃣ fail – «Ой, что‑то пошло не так…» 😵‍💫

```json
{
  "uuid": "123e4567-e89b-12d3-a456-426614174000",
  "payload": {
    "typ": "fail",
    "err": "Не удалось разобрать JSON"
  }
}
```

- **Генерируется автоматически**, если сервер не смог прочитать присланную JSON‑строку.
- **Работает как `done`**: удаляет сообщение из хранилища, но вместо радости – ошибка.

# 🔌 Как подключиться? (очень просто)

## Через WebSocket (для браузера, мобилки, IoT)

```javascript
const ws = new WebSocket("ws://localhost:8000/любой_путь");

ws.onopen = () => {
  console.log("🔗 Влетели!");
  ws.send(JSON.stringify({
    uuid: crypto.randomUUID(),
    payload: { typ: "send", top: "привет", mes: "Кто тут?" }
  }));
};

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  if (msg.payload.typ === "send") {
    console.log(`📨 ${msg.payload.top}: ${msg.payload.mes}`);
    // подтверждаем получение
    ws.send(JSON.stringify({
      uuid: msg.uuid,
      payload: { typ: "done" }
    }));
  }
};
```

> **Важно:** как только открыли соединение – сервер сразу высылает **все старые неподтверждённые сообщения**. Будь готов! 📚

## Через HTTP POST (для бэкендов, скриптов)

```bash
curl -X POST http://localhost:8000/api/push \
  -H "Content-Type: application/json" \
  -d '{
    "uuid": "123e4567-e89b-12d3-a456-426614174000",
    "payload": {
      "typ": "send",
      "top": "статус",
      "mes": "Деплой удался ✅"
    }
  }'
```

- Ответ: `201 Created` и тело – твоё же сообщение (уже разосланное).
- Через POST можно отправлять и `done` тоже.

# 🎯 Какие задачи можно решать с MessageCenter?

| Задача | 💡 Как это работает |
|--------|----------------------|
| **Чат с гарантией доставки** | Отправил `send` – все сразу видят. Когда прочитали – кидают `done`. Новый участник получает все непрочитанные сообщения. |
| **Уведомления для фронтенда** | Сервер шлёт `send` через POST, а все открытые веб‑морды получают всплывашки. |
| **Очередь команд для ботов/устройств** | Команда сохраняется до тех пор, пока устройство не подтвердит выполнение. Если устройство перезагрузилось – подключается и получает всё, что не сделано. |
| **Логи в реальном времени** | Сервер логирует события и отправляет их всем, кто подписан. |
| **Синхронизация между микросервисами** | Один сервис отправляет событие через POST, другие получают через WebSocket и реагируют мгновенно. |

# 🔥 Фишки, которые делают систему прикольной

- 🧠 **Умное хранилище** – DiskCache на диске, не теряет данные даже после падения.
- ⚡ **Асинхронность до мозга костей** – много клиентов не тормозят друг друга.
- 🧩 **Схема сообщений** – чёткая, простая, на Pydantic (валидация на уровне ядра).
- 🧹 **Самозачистка** – мёртвые WebSocket‑соединения вылетают из пула автоматически.
- 📡 **Любой путь** – не важно, какой URL для WebSocket или POST, всё работает.
- 🧪 **Отказоустойчивость** – если при отправке клиенту соединение оборвалось, клиент удаляется из рассылки.

# 🧙‍♂️ Коротко о главном (для самых быстрых)

1. Подключаешься по WebSocket → получаешь все висящие `notify`.
2. Шлёшь `send` → сервер сохраняет и кидает всем.
3. Получил уведомление → отправь `done` с тем же `uuid`.
4. Нет `done` → сообщение будет жить вечно и приходить каждому новому.
5. Можно слать сообщения и через обычный HTTP POST.

## 🎁 Бонус: пример жизни сообщения

1. **Клиент А** отправляет `send` на тему «котики».
2. Сервер сохраняет это сообщение в кеш и **мгновенно** отдаёт всем (включая клиента А).
3. **Клиент Б** подключается через 5 минут – сервер отправляет ему то же сообщение (потому что `done` ещё не пришло).
4. **Клиент Б** обрабатывает и шлёт `done`.
5. Сервер удаляет запись о «котиках» – больше это сообщение никому не придёт.
6. **Клиент В** подключается через час – он уже не увидит этого сообщения, потому что оно подтверждено.
