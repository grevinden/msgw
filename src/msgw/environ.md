В файле `environ.py` представлена конфигурация приложения через вложенные модели Pydantic. Благодаря `env_nested_delimiter = '_'` и структуре классов, переменные окружения формируются по шаблону `PREFIX_CLASS_FIELD`.

**Глобальный префикс:** `MSGW_` (если не задана переменная `APP`).

Ниже приведён полный список переменных окружения, сгруппированный по логическим блокам.

### 1. Кэш (`cache`)
Отвечает за подключение к хранилищу состояний (Redis, MongoDB или память).

| Переменная       | Тип         | По умолчанию | Описание                                                                                                           |
|:-----------------|:------------|:-------------|:-------------------------------------------------------------------------------------------------------------------|
| `MSGW_CACHE_URL` | `str` (URL) | `mem://`     | URL подключения. Поддерживаемые схемы: `mem://`, `redis://`, `mongo://`. <br> *Пример:* `redis://localhost:6379/0` |
| `MSGW_CACHE_TTL` | `int`       | `3600`       | Время жизни сообщения в секундах (по умолчанию).                                                                   |

> **Важно:** В методе `model_post_init` для схемы `redis` автоматически добавляются параметры `pickle_type=null` и `client_side=True`.

### 2. Шифрование ECIES (`ecies`)
Управляет режимом Reverse Proxy с расшифровкой токенов.

| Переменная       | Тип   | По умолчанию | Описание                                                                                                                                                  |
|:-----------------|:------|:-------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------|
| `MSGW_ECIES_KEY` | `str` | `null`       | Приватный ключ для расшифровки. **Строгий формат:** ровно 43 символа Base64Url (без знаков `=`). <br> *Если не задан, прокси-маршруты не регистрируются.* |

*Доступны также вычисляемые поля модели (не задаются через env, только для чтения внутри приложения):*
*   `ecies.bytes` — декодированные байты ключа (32 байта).
*   `ecies.enabled` — `true`, если ключ задан.

### 3. Системные пути (`path`)
Служебная информация о расположении файлов.

| Переменная       | Тип          | Описание                                                                   |
|:-----------------|:-------------|:---------------------------------------------------------------------------|
| `MSGW_PATH_ROOT` | `str` (Path) | Абсолютный путь к корневой директории проекта (вычисляется автоматически). |

> **Примечание:** Параметры health-чека (`timeout=2`, `interval=3`) **хардкодятся** в `proxy.py` и не настраиваются через переменные окружения.

---

### Пример `.env` файла для запуска с Redis

```env
# Cache Configuration
MSGW_CACHE_URL=redis://redis:6379/0
MSGW_CACHE_TTL=7200

# ECIES Proxy Configuration
# Сгенерирован через: wg genkey | basenc --base64url -w0 | tr -d '='
MSGW_ECIES_KEY=uhgE8myhNkWOmYN78X906cUkB_B4PBOcRJoGIlg-UQU
```

### Примечание по именованию
В классе `Settings` используется `env_nested_delimiter = '_'`. Это означает, что Pydantic ожидает переменные в формате `PREFIX_CLASS_FIELD`, где `CLASS` — имя вложенной модели, а `FIELD` — имя поля. Например:
- `MSGW_CACHE_URL` → модель `Cache`, поле `url`
- `MSGW_ECIES_KEY` → модель `Ecies`, поле `key`

Если переменные выше не подхватываются, убедитесь, что используется версия `pydantic-settings >= 2.0` и префикс совпадает с содержимым переменной `APP` (по умолчанию `MSGW`).
